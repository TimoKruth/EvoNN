Immediate foundation

Make promoted evidence the default review surface

Every serious compare run should be promoted into the registry, then reviewed from the registry report rather than from transient workspaces.

  • Add git commit, branch, backend, hardware, output-quality level, and contender-floor state into registry rows when available.
  • Connect the canonical dashboard directly to registry rows, not only copied fair-matrix summaries.
  • Add `evidence/README.md` with retention, promotion, and cleanup rules.
  • Add stale-artifact and missing-artifact warnings to `evidence validate`.

Decision loop

Run repeated Tier B and Tier C evidence campaigns

The next useful comparison work is not broader benchmark breadth. It is repeated evidence on the tiers that already expose engine differences.

  • Run `tier_b_core_v2` at budget `96` with three seeds.
  • Run `tier_b_core_v2` at budget `384` with two seeds if the low-budget signal is meaningful.

  • Promote all clean summaries into one registry label and inspect decision blockers.

Research proof

Prove or reject Primordia to Topograph transfer

This is the most important scientific gap. The system needs to show whether primitive motifs discovered by Primordia actually improve Topograph under fair accounting.

  • Harden the seed artifact contract with target compatibility, checksum, source budget, descriptor, and contamination policy.
  • Make Topograph's native seed ingestion visibly change initialization or search bias.
  • Run unseeded, directly seeded, and staged-seeded variants with identical pack, budget, seed, and backend.
  • Classify the result as gain, no material change, regression, or inconclusive in the evidence registry.
Engine quality

Use evidence to split engine work by role

Avoid making all engines weak variants of the same loop. Each engine should earn a role from repeated evidence.

  • Prism: improve budget allocation, family diagnostics, and Pareto scoring.
  • Topograph: add topology descriptors, quality-diversity archive evidence, and seed ingestion ablations.
  • Stratograph: diagnose LM flatline behavior and prove whether hierarchy adds value.
  • Primordia: focus on motif bank quality, seed-source usefulness, and real LM/image validation.
  • Contenders: keep the floor strong enough that EvoNN wins mean something.
L4 evidence

Upgrade decision labels before optimizing performance

Performance work is valuable, but only after the comparison layer can distinguish real gains from noisy wins.

  • Add before/after cohort comparisons with explicit regression and likely-gain labels.
  • Add effect sizes against best required contender and against previous engine baseline.
  • Make ceiling ties non-promotional evidence.
  • Detect backend drift, mixed-budget ambiguity, and missing repeated seeds as blockers.

Do not prioritize

Things to avoid now

  • Adding many new benchmark tiers before Tier B/C repeated evidence is stable.
  • Claiming superiority from single-seed broad-lane runs.
  • Creating new markdown-only plans without executable gates.
  • Doing large performance refactors before budget truth and L4 labels are stronger.
